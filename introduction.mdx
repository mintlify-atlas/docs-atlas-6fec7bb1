Lantern is a powerful service designed to effortlessly broadcast your active Discord status to both a RESTful API endpoint and a WebSocket connection. Whether you want to showcase your current Spotify tracks on your personal site or display your Discord activities in real-time, Lantern has you covered.

## What is Lantern?

Lantern monitors your Discord presence and makes it available through two interfaces:

- **RESTful API**: Access user data at `lantern.rest/api/v1/users/:user_id`
- **WebSocket**: Real-time updates via `wss://lantern.rest/socket`

While Lantern is ready to use out-of-the-box without any deployment, it also offers the flexibility for self-hosting with minimal setup.

## What data does Lantern provide?

Lantern provides comprehensive user data including:

- **User metadata**: Username, discriminator, global name, avatar, badges, and Discord flags
- **Active platforms**: Current status on desktop, mobile, and web
- **Spotify integration**: Currently playing track, artist, album, album cover, and playback timestamps
- **Activities**: Games, custom statuses, and rich presence data with application details
- **Server tag**: Guild badges and server affiliations
- **Last seen**: Timestamp of when the user was last online (when offline)
- **Key-value storage**: Custom data stored by the user

## Getting started

To start using Lantern, you'll need:

1. A Discord user ID to monitor (17-19 digit number)
2. The user must be monitored by the Lantern system (join the Lantern Discord server)

<Note>
Users must be part of the Lantern Discord server to be monitored. Once joined, their presence data becomes available through the API and WebSocket.
</Note>

Check out the [quickstart guide](/quickstart) to make your first API call and establish a WebSocket connection.
