## Get users by IDs

<CodeGroup>
```bash cURL
curl "https://lantern.rest/api/v1/users?user_ids=123456789012345678&user_ids=234567890123456789"
```

```javascript JavaScript
const response = await fetch(
  'https://lantern.rest/api/v1/users?user_ids=123456789012345678&user_ids=234567890123456789'
);
const users = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://lantern.rest/api/v1/users',
    params={'user_ids': ['123456789012345678', '234567890123456789']}
)
users = response.json()
```
</CodeGroup>

<ParamField query="user_ids" type="array" required>
  An array of user IDs to fetch data for. Each ID must be a string of 17-19 characters, and there must be no duplicates.
  
  **Validation rules:**
  - Minimum 1 item
  - Maximum 50 items
  - Must be unique (no duplicates)
  - Each ID: 17-19 characters, numeric only
  - Pattern: `^[0-9]{17,19}$`
</ParamField>

<Note>
Include the `user_ids` parameter multiple times for each user ID you want to retrieve. For example: `?user_ids=ID1&user_ids=ID2`
</Note>

## Get user by ID

<CodeGroup>
```bash cURL
curl "https://lantern.rest/api/v1/users/123456789012345678"
```

```bash SVG Badge
curl "https://lantern.rest/api/v1/users/123456789012345678?svg=1&theme=dark&borderRadius=2"
```

```javascript JavaScript
const response = await fetch(
  'https://lantern.rest/api/v1/users/123456789012345678'
);
const user = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://lantern.rest/api/v1/users/123456789012345678'
)
user = response.json()
```
</CodeGroup>

### Path parameters

<ParamField path="id" type="string" required>
  The Discord user ID to fetch data for
  
  **Validation:**
  - Must be numeric
  - Length: 17-19 characters
  - Pattern: `^[0-9]{17,19}$`
</ParamField>

### Query parameters

<ParamField query="svg" type="integer" default={0}>
  Whether to return the user's data as an SVG image instead of JSON
  
  **Values:** `0` or `1`
</ParamField>

<ParamField query="theme" type="string" default="dark">
  The theme to use for the SVG image (only applies when `svg=1`)
  
  **Values:** `light` or `dark`
</ParamField>

<ParamField query="borderRadius" type="integer" default={2}>
  The border radius (in rem units) for the SVG image
  
  **Validation:**
  - Must be numeric
  - Minimum: 0
  - Pattern: `^[0-9]+$`
</ParamField>

<ParamField query="hideGlobalName" type="integer" default={0}>
  Whether to hide the user's global name in the SVG
  
  **Values:** `0` or `1`
</ParamField>

<ParamField query="hideStatus" type="integer" default={0}>
  Whether to hide the user's status in the SVG
  
  **Values:** `0` or `1`
</ParamField>

<ParamField query="hideBadges" type="integer" default={0}>
  Whether to hide the user's badges in the SVG
  
  **Values:** `0` or `1`
</ParamField>

<ParamField query="hideActivity" type="integer" default={0}>
  Whether to hide the user's activity in the SVG
  
  **Values:** `0` or `1`
</ParamField>

<ParamField query="hideLastSeen" type="integer" default={0}>
  Whether to hide the user's last seen time in the SVG
  
  **Values:** `0` or `1`
</ParamField>

<ParamField query="hideServerTag" type="integer" default={0}>
  Whether to hide the user's server tag in the SVG
  
  **Values:** `0` or `1`
</ParamField>

<ParamField query="noActivityTitle" type="string" default="No activity">
  The title to display when the user has no activity
  
  **Validation:**
  - Minimum length: 1 character
  - Maximum length: 64 characters
</ParamField>

<ParamField query="noActivityMessage" type="string" default="User is not currently doing anything.">
  The message to display when the user has no activity
  
  **Validation:**
  - Minimum length: 1 character
  - Maximum length: 256 characters
</ParamField>

### Response

When `svg=0` (default), returns the same user object structure as the bulk endpoint above.

When `svg=1`, returns an SVG image with content type `image/svg+xml`.

### Error responses

  ```json
  }
  ```
