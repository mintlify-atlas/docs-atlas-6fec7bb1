Lantern provides a simple key-value (KV) storage system that lets you store custom data for users. Access storage through the REST API, WebSocket, or the Lantern Discord bot.

## Use cases

The KV storage system is perfect for:

- **Configuration values** - Store user preferences, theme settings, or feature flags for your website
- **Dynamic profile data** - Current location, custom status messages, or bio information
- **User-specific data** - Preferences, settings, or any custom metadata
- **Temporary data** - Cache values, session data, or temporary states

## Storage limits

<Note>
  **Keys**
  - Must be alphanumeric only (a-z, A-Z, 0-9)
  - Maximum 255 characters
  - No special characters or spaces

  **Values**
  - Can be any string (including JSON)
  - Maximum 30,000 characters per value
  - Store JSON objects by stringifying them

  **Total storage**
  - Maximum 512 key-value pairs per user
</Note>

From the source code validation:

```typescript
kv: {
  type: Map,
  of: String,
  validate: {
    validator(v: Map<string, string>) {
      // Ensure that each key is alphanumeric and doesn't exceed 255 characters
      for (const key of v.keys()) {
        if (!/^[a-zA-Z0-9]+$/.test(key) || key.length > 255) {
          return false;
        }
      }

      // Ensure that each value doesn't exceed 30,000 characters
      for (const value of v.values()) {
        if (value.length > 30000) {
          return false;
        }
      }

      return true;
    }
  }
}
```

## Authentication

Most KV storage operations require authentication using an authorization token.

### Getting your token

Use the Lantern Discord bot to generate your token:

```
/storage create token
```

The bot will provide a unique token. Keep this secure - it grants full access to your storage.

### Using the token

Include the token in the `Authorization` header for all authenticated requests:

```bash
curl -H "Authorization: your_token_here" \
  https://lantern.rest/api/v1/users/123456789012345678/storage
```

<Warning>
  Without the `Authorization` header, you can only read public storage data. All write operations require authentication.
</Warning>

## REST API operations

### Get all key-value pairs

Retrieve all storage entries for a user:

```http

```

| Name | Type | Description |
| --- | --- | --- |
| `user_id` | string | The Discord user ID |

```json
}
```

### Create a key-value pair

Create a new storage entry:

```http
PUT /api/v1/users/:user_id/storage/:key
```

<Warning>
  Requires `Authorization` header
</Warning>

#### Parameters

| Name | Type | Description |
| --- | --- | --- |
| `user_id` | string | The Discord user ID |
| `key` | string | The key to create (alphanumeric, max 255 chars) |
| `value` | string | The value to store (max 30,000 chars) |

#### Request body

```json
{
  "value": "San Francisco, CA"
}
```

#### Response

```json
{
  "success": true
}
```

<CodeGroup>
```bash cURL
curl -X PUT \
  -H "Authorization: your_token_here" \
  -H "Content-Type: application/json" \
  -d '{"value":"San Francisco, CA"}' \
  https://lantern.rest/api/v1/users/123456789012345678/storage/location
```

```javascript JavaScript
const response = await fetch(
  'https://lantern.rest/api/v1/users/123456789012345678/storage/location',
  {
    method: 'PUT',
    headers: {
      'Authorization': 'your_token_here',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ value: 'San Francisco, CA' })
  }
);
const result = await response.json();
console.log(result.success); // true
```

```python Python
import requests

response = requests.put(
  'https://lantern.rest/api/v1/users/123456789012345678/storage/location',
  headers={'Authorization': 'your_token_here'},
  json={'value': 'San Francisco, CA'}
)
print(response.json())
```
</CodeGroup>

## WebSocket integration

When connected to the WebSocket, you'll automatically receive storage updates for subscribed users.

### Opcode 10: Storage update

Received whenever a subscribed user's storage is modified:

```json
{
  "t": "STORAGE_UPDATE",
  "op": 10,
  "d": {
    "location": "San Francisco, CA",
    "website": "https://example.com",
    "theme": "dark"
  }
}
```

<Info>
  The `d` field contains **all** key-value pairs for the user, not just the updated ones. This ensures your client always has the complete storage state.
</Info>

### Example WebSocket handler

```javascript
ws.on('message', (data) => {
  const message = JSON.parse(data);
  const { op, d } = message;

  if (op === 10) { // STORAGE_UPDATE
    console.log('Storage updated:', d);
    
    // Update your UI with the new storage data
    if (d.theme) {
      updateTheme(d.theme);
    }
    
    if (d.location) {
      updateLocation(d.location);
    }
  }
});
```

From the source implementation, storage updates are sent automatically:

```typescript
Model.watch().on('change', async (metadata) => {
  const foundStorage = await Model.findById(_id).lean();
  
  // Send a message to all active sockets that the user's storage has changed
  for (const [, data] of ActiveSockets) {
    if (data.subscribed === 'ALL' || data.subscribed.includes(foundStorage.userId)) {
      socket_send(data.instance, STORAGE_UPDATE, createUserData(foundStorage.userId, foundStorage.kv));
    }
  }
});
```

## Storing JSON data

Since values must be strings, store complex data by stringifying JSON:

<CodeGroup>
```javascript Create JSON storage
const config = {
  notifications: true,
  language: 'en',
  fontSize: 14
};

await fetch(
  'https://lantern.rest/api/v1/users/123456789012345678/storage/config',
  {
    method: 'PUT',
    headers: {
      'Authorization': 'your_token_here',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ value: JSON.stringify(config) })
  }
);
```

```javascript Retrieve JSON storage
const response = await fetch(
  'https://lantern.rest/api/v1/users/123456789012345678/storage/config'
);
const data = await response.json();
const config = JSON.parse(data.value);

console.log(config.notifications); // true
console.log(config.language); // "en"
```
</CodeGroup>

## Use case examples

### Personal website configuration

Store theme and preferences:

```javascript
// Set website preferences
await Promise.all([
  setStorage('theme', 'dark'),
  setStorage('accentColor', '#3b82f6'),
  setStorage('showSpotify', 'true'),
  setStorage('showActivity', 'true')
]);

// Retrieve on page load
const storage = await getStorage();
applyTheme(storage.theme);
setAccentColor(storage.accentColor);
toggleSpotify(storage.showSpotify === 'true');
```

### Dynamic profile information

Store frequently changing data:

```javascript
// Update current location
await setStorage('location', 'Seattle, WA');

// Update status message
await setStorage('status', 'Building something cool');

// Update custom links
const links = JSON.stringify([
  { name: 'GitHub', url: 'https://github.com/username' },
  { name: 'Twitter', url: 'https://twitter.com/username' }
]);
await setStorage('links', links);
```

### Feature flags

Control feature visibility:

```javascript
// Enable beta features
await setStorage('betaAccess', 'true');
await setStorage('newDesign', 'true');
await setStorage('experimentalFeatures', 'analytics,comments');

// Check feature flags
const storage = await getStorage();
const hasBetaAccess = storage.betaAccess === 'true';
const features = storage.experimentalFeatures?.split(',') || [];
```

## Best practices

<Accordion title="Keep keys descriptive">
  Use clear, descriptive key names like `websiteTheme` instead of `t1`. This makes your storage easier to debug and maintain.
</Accordion>

<Accordion title="Validate before storing">
  Always validate data before storing, especially when accepting user input. Ensure values stay within the 30,000 character limit.
</Accordion>

<Accordion title="Use JSON for complex data">
  For structured data, stringify JSON objects. This lets you store arrays, nested objects, and complex configurations in a single key.
</Accordion>

<Accordion title="Handle missing keys gracefully">
  Always provide fallback values when retrieving storage data. Keys may not exist if they haven't been set yet.
  
  ```javascript
  const theme = storage.theme || 'light'; // Default to 'light'
  ```
</Accordion>

<Accordion title="Secure your token">
  Never expose your authorization token in client-side code. Make API calls from your backend server, or use environment variables for security.
</Accordion>

<Accordion title="Monitor storage usage">
  Keep track of how many keys you're using. With a limit of 512 pairs, it's easy to hit the cap if you're storing data dynamically.
</Accordion>
